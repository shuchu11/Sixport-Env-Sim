# 六埠（Six-Port）CW 雷達多徑干擾模擬器 — 研究筆記

> 目標：建立一個能模擬實際環境、採集實驗數據的模擬器，用來研究六埠雷達的**多徑（multipath）干擾抑制**方法。

---

## 1. 背景：為什麼六埠雷達沒有現成的「環境模擬器」

毫米波 FMCW 雷達有不少現成的場景模擬器（RadarSimPy、ViRa、RadaRays 等），但六埠雷達幾乎沒有。原因在於六埠雷達的核心不在「場景」，而在 **RF 前端**：

- 六埠接收機僅用被動元件（方向耦合器、功率分配器、90° hybrid）搭配四個功率感測器，直接在微波/毫米波頻段萃取相對相位與相對振幅，省去傳統 I/Q 解調晶片。
- 硬體的非理想特性需要靠校正流程消除。
- 因此「模擬六埠雷達」＝在 RF 網路層級自己搭，重點是把檢波 → I/Q → 校正這條鏈做對。

**結論：與其找套件，不如直接做一個針對研究目的的模擬器。**

## 2. 物理模型

### 2.1 總回波（多徑如何進入系統）

多條傳播路徑的複數回波相加成總回波：

```
B(t) = Σ_k  ρ_k · exp( j · φ_k(t) )
φ_k(t) = (4π / λ) · ( d_k + x_k(t) ) + φ_k0
```

- `ρ_k`：第 k 條路徑的反射率（含 RCS、路徑損耗、天線增益）
- `d_k`：標稱單程距離（來回相位用 4π/λ）
- `x_k(t)`：該路徑反射體的位移時間序列
- 「相加」這個動作，**正是多徑/多目標干擾進入系統的數學本質**

典型三條路徑的研究場景：

| 路徑 | 角色 | 運動 |
|------|------|------|
| chest | 主目標（胸腔） | 呼吸 0.3 Hz + 心跳 1.2 Hz |
| wall | 靜態多徑（牆） | 無（固定相量） |
| fan | 動態多徑（風扇/第二人） | 正弦振動 0.75 Hz |

### 2.2 六埠檢波

四個檢波器各量測一路功率（理想參考相位 0/90/180/270°）：

```
P_i(t) = | B(t) + L·exp(j·θ_i) |²
V_i(t) = R_i · P_i + γ_i · P_i² + DC_i + noise
```

差分組合還原 I/Q：

```
I = v1 − v3      (θ = 0° vs 180°)
Q = v2 − v4      (θ = 90° vs 270°)
```

理想下差分可消掉共同的 `|B|² + L²` 項。

### 2.3 位移還原

```
φ(t) = unwrap( atan2(Q, I) )
x(t) = φ(t) · λ / (4π)        （CW 只觀測得到相對位移，需扣均值）
```

### 2.4 可掃描的硬體非理想性

模擬器刻意把「會破壞訊號的東西」全開放成參數：

- **增益失衡**：四個檢波器響應 `R_i` 不匹配
- **二階非線性**：`γ_i · P²`，差分後無法完全對消 → 失真
- **埠相位誤差**：四個 `θ_i` 不剛好是 0/90/180/270°
- **DC 偏移**：LO–RF 洩漏 / 偏壓
- **加性雜訊**：熱雜訊 / 閃爍雜訊

## 3. 怎麼採集實驗數據

最小流程＝「擺場景 → 收數據」：

```python
from sixport_radar_sim import (
    SixPortRadarSimulator, Path, vital_signs_motion, sinusoid_motion
)

sim = SixPortRadarSimulator(f0_hz=24e9, fs_hz=200.0)
sim.add_path(Path(0.50, 1.0, vital_signs_motion(),          name="chest"))  # 主目標
sim.add_path(Path(0.87, 0.8,                                 name="wall"))   # 靜態多徑
sim.add_path(Path(0.70, 0.5, sinusoid_motion(2e-3, 0.75),    name="fan"))    # 動態干擾

data = sim.collect(duration_s=30)        # data["V"] 是 4×N 的量測數據
SixPortRadarSimulator.save(data, "exp01.npz", "exp01.csv")
```

- `sixport_dataset.csv`：30 秒 × 200 Hz，欄位 `t, v1, v2, v3, v4`，可直接餵演算法
- `.npz` 版額外存 ground truth，方便算 RMSE
- 想換載波（5.8 GHz ISM / 24 GHz / 60 GHz mmWave）、採樣率、硬體缺陷，全部改參數即可

## 4. 關鍵研究結論（從 demo 看出）

`sixport_demo.png` 的 I/Q 平面把問題講得很清楚：

- **乾淨場景**：I/Q 是一段乾淨的弧。
- **加多徑後**：整段被推離原點、被擠壓變形。
- **靜態多徑**：等於在 I/Q 平面加一個固定相量 → 把圓心平移（demo 量到圓心偏移 ≈ (−14.4, 2.4)）。

位移還原 RMSE（對 ground truth）：

| 情況 | RMSE (mm) |
|------|-----------|
| 乾淨場景，校正後 | **0.016** |
| 多徑，無抑制 | **8.36** |
| 多徑，橢圓校正後 | **3.06** |

**核心洞察：**

> 橢圓校正能吃掉**靜態多徑**（移除 DC 偏移 + 還原圓形），
> 但**動態多徑**（移動干擾源）因為自己也在轉相量，校正吃不掉，殘差就卡在這裡。
> 這就是六埠雷達多徑問題真正困難、也最值得發論文的地方。

## 5. 你的干擾抑制研究路線圖

模擬器已寫好基線（橢圓校正）。你的方法只要在 `extract_iq` 之後接手，跟基線比 RMSE 即可。

### 5.1 靜態雜波校正（較成熟）

- 基線的橢圓/弧線擬合已相當有效
- 可研究方向：弧段很短（強靜態回波壓過弱目標）時更穩健的圓心估計
- 具體方法：arc-fitting、RANSAC 抗離群點

### 5.2 動態干擾分離（硬骨頭，研究價值高）

單通道 CW 六埠**沒有距離解析力**，分離只能靠運動的頻率/都卜勒差異：

- **譜域濾波**：呼吸 0.3 Hz、心跳 1.2 Hz、干擾 0.75 Hz 在頻譜上是分開的
- **自適應濾波**：LMS / RLS 對消已知干擾頻率
- **盲源分離**：EMD、VMD、ICA

### 5.3 波形/系統層面（最根本）

- CW → 步進頻率 / FMCW，取得距離維度
- 多天線增加空間自由度，把不同路徑分開
